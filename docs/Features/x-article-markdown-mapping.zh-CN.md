# Markdown → X 文章：格式支持一览

Kaitox 把你的 Markdown 变成 X（Twitter）文章时，哪些元素能「原样过关」、哪些会降级、哪些会丢——这份表就是答案。

X 文章**不是** Markdown：编辑器存的是 Draft.js 文档（blocks + entities），不是文本。转换器把每个 Markdown 构造映射到最接近的 X 原语：大多数干净落地，少数降级，个别被丢弃。下表全部由 `@kaitox/x-article` 里的 [`markdownToContentState`](../../packages/x-article/README.zh-CN.md) 产出，并与它的样式检查器对齐——每个入口（agent skill、CLI、Obsidian、扩展）跑的都是同一套引擎，所以无论从哪推，结果都一致。

## 图例

| | 含义 |
|---|---|
| ✅ **支持** | 干净映射，无损失。 |
| 🟡 **降级** | 能上传，但会丢一部分保真度（见「说明」）。样式检查器标为 `info` / `warning`。 |
| ❌ **丢弃** | 内容丢失或显示成乱码。样式检查器标为 `warning` / `error`。 |

## 块级元素

| Markdown | 在 X 里变成 | | 说明 |
|---|---|---|---|
| 段落 | 段落 | ✅ | |
| 第一个 `# H1` | **文章主标题** | ✅ | 被当作标题吃掉——不出现在正文里。Setext（`===`）H1 也认。 |
| 第 2 个及以后的 `# H1` | Heading | 🟡 | 只有第一个 H1 是标题；多出来的 H1 按 `##` 渲染。一篇留一个。 |
| `## H2` | Heading | ✅ | |
| `### H3` | SubHeading | ✅ | |
| `#### H4` 及更深 | SubHeading | 🟡 | 被钳住——X 正文只有两级标题。 |
| `> 引用` | Blockquote | ✅ | 引用里每个段落输出一块。 |
| `- ` / `* ` 无序列表 | 无序列表 | ✅ | 每项一块。 |
| `1. ` 有序列表 | 有序列表 | ✅ | 每项一块。 |
| 嵌套列表子项 | _(被拍平丢弃)_ | 🟡 | 子项会**被静默丢弃**。请拍平成一级。 |
| `- [ ]` 任务列表 | 无序列表 | 🟡 | 勾选框丢失，文字保留。 |
| ` ``` ` 围栏代码 | 代码框 | ✅ | 纯文本——无语法高亮。 |
| ` ```mermaid ` 围栏 | 图表**图片** | ✅ | 上传时渲染成 PNG（需要浏览器扩展）。mermaid 语法错误会导致上传失败——先确认图能画出来。 |
| 表格 | 表格 | ✅ | X 原生渲染。 |
| `---` 分割线 | Divider | ✅ | |
| `![alt](src)` | 图片 | ✅ | 需要图片字节 → 上传换取 `media_id`。解析不到的 `src` 会被跳过（文章其余部分照常上传）。 |
| **独占一行**的 x.com/twitter 帖子链接 | 内嵌引用推文 | ✅ | 仅当链接独占一行时触发。同样的 URL 夹在句中则只是普通链接。 |
| 裸 HTML 块（`<div>…`） | _(丢弃)_ | ❌ | 整块被丢掉。改写成 Markdown。 |

## 行内 / 文字格式

| Markdown | 在 X 里变成 | | 说明 |
|---|---|---|---|
| `**加粗**` | **加粗** | ✅ | |
| `*斜体*` / `_斜体_` | _斜体_ | ✅ | |
| `~~删除线~~` | ~~删除线~~ | ✅ | |
| `[文字](url)` | 链接 | ✅ | |
| `[文字][ref]` + 定义 | 链接 | ✅ | 引用式链接会被解析。 |
| 裸 URL `https://…` | 链接 | ✅ | 自动成链（GFM）。 |
| `` `行内代码` `` | 纯文本 | 🟡 | X 文章没有行内代码样式——文字保留，等宽样式丢失。 |
| 硬换行 | 换行 | ✅ | 行尾两个空格、行尾 `\`、或 `<br>` → 块**内**换行。 |
| 行内 `<b>…</b>` HTML | 字面文本 | ❌ | 标签原样显示，不渲染。 |
| 脚注 `[^1]` | 乱码文本 | ❌ | 不按脚注解析——标记会漏进正文。请把注释内联，或改成普通链接。 |
| 段落中间的行内图片 | 把段落切开 | ✅ | 图片单独成块（X 里图片一律块级）；它两侧的文字拆成各自的段落。 |

## Frontmatter

YAML frontmatter 只读两个字段，读完剥掉——永远不进正文。

| 字段 | 作用 | |
|---|---|---|
| `title:` | 设置文章标题。覆盖第一个 `# H1`。 | ✅ |
| `cover:` | 封面图（本地 / vault 路径）；推送端解析成字节，在草稿创建后设上去。 | ✅ |

其余 frontmatter 键一律忽略。**标题优先级：** frontmatter `title:`（或 CLI `--title` 参数）› 第一个 `# H1` › 文件名。

## 不支持的，一眼看清

- ❌ **裸 HTML**——块级 HTML 被丢弃；行内标签显示成字面文本。
- ❌ **脚注**（`[^1]`）——以乱码标记漏进正文。
- 🟡 **嵌套列表**——只保留最外一级，子项消失。
- 🟡 **行内代码**——渲染成纯文本（无等宽）。
- 🟡 **任务列表勾选框**、**第三级以上标题**、**多余的 H1**——按上文降级。

当你保留 🟡/❌ 构造并选择「纯文本」兜底时，只有真正会丢内容的才会被改写（HTML 去标签成文本、嵌套列表拍平）；表格和代码围栏保持原样，因为 X 对它们渲染得很好。

## 注意与坑

- **标题只有两级。** `##` → Heading，`###` → SubHeading；`#` 是标题（从正文移除）；`####`+ 钳到 SubHeading。X 文章正文就是没有更深的层级。
- **内嵌只认「独占一行」。** 帖子链接只有独占一行时才变成内嵌引用——和 X 自己「粘贴即嵌入」的行为一致。一行一条，即一行一个内嵌。
- **图片以字节传输，不是 URL。** 推送端打包本地图片字节，扩展在你已登录的会话里上传换取 `media_id`。远程（`http(s)`）图片不会被自动抓取——自定义上传端需先把它下载进草稿包。
- **图片超大通常没事。** relay 入库时会自动重压超限的 PNG / JPEG / WebP。只有不可压缩格式（GIF、SVG）超过约 5 MB 才可能被 X 拒。
- **偏移量是 UTF-16 code unit。** 一个 CJK 字符按 1 计，多数 emoji 按 2 计。这只在你自己构造 `content_state` 时才需要关心——转换器已经处理好了。
